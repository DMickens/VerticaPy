vDataFrame.to_csv¶

In [ ]:
vDataFrame.to_csv(name: str,
                  path: str = '', 
                  sep: str = ',',
                  na_rep: str = '',
                  quotechar: str = '"',
                  usecols: list = [],
                  header: bool = True,
                  new_header: list = [],
                  order_by = [],
                  limit: int = 0)

Creates a CSV file of the current vDataFrame relation.

Parameters¶

Name Type Optional Description
name
str
❌
 Name of the CSV file. Be careful: if a CSV file with the same name exists, it will over-write it.
path
str
✓
 Absolute path where the CSV file will be created.
sep
str
✓
 Column separator.
na_rep
str
✓
 Missing values representation.
quotechar
str
✓
 Char which will enclose the str values.
usecols
list
✓
 vcolumns to select from the final vDataFrame relation. If empty, all the vcolumns will be selected.
header
bool
✓
 If set to False, no header will be written in the CSV file.
new_header
list
✓
 List of columns used to replace vcolumns name in the CSV.
order_by
dict / list
✓
 List of the vcolumns used to sort the data using asc order or dictionary of all the sorting methods. For example, to sort by "column1" ASC and "column2" DESC, write {"column1": "asc", "column2": "desc"}
limit
int
✓
 If greater than 0, the maximum number of elements to write at the same time in the CSV file. It can be used to minimize memory impacts. Be sure to keep the same order to avoid unexpected results.

Returns¶

vDataFrame : self
