Let's create a Logistic Regression to predict the survival of the Titanic passengers. We will use the age and the fare as predictors.

In [3]:
from verticapy.learn.linear_model import LogisticRegression
model = LogisticRegression("public.LR_titanic")
model.fit("public.titanic", ["age", "fare"], "survived")

By fitting the model, new model's attributes will be created. These attributes will be used to simplify the methods usage.

Out[6]:

In [7]:
model.test_relation
Out[7]:
'public.titanic'

In our case, we did not write the test relation when fitting the model. The model will then consider the train relation as test. These attributes will be used when invoking the different model abstractions. For example, let's compute the accuracy of the model.

In [8]:
model.score(method = "accuracy")
Out[8]:
0.6837349397590361

The 'score' method is using the attribute 'y' and the model prediction in the 'test_relation' to compute the accuracy. It is then possible to change them anytime to deploy the models on different columns. The model could also have other useful attributes. In the case of Logistic Regression, the 'coef' attribute can give you useful information.
