DBSCAN (Beta)¶

In [ ]:
DBSCAN(name: str,
       cursor = None,
       eps: float = 0.5,
       min_samples: int = 5,
       p: int = 2)

Creates a DBSCAN object by using the DBSCAN algorithm as defined by Martin Ester, Hans-Peter Kriegel, JÃ¶rg Sander and Xiaowei Xu. This object is using pure SQL to compute all the distances and neighbors. It is also using Python to compute the cluster propagation (non scalable phase). This model is using CROSS JOIN and may be really expensive in some cases. It will index all the elements of the table in order to be optimal (the CROSS JOIN will happen only with IDs which are integers). As DBSCAN is using the p-distance, it is highly sensible to un-normalized data. However, DBSCAN is really robust to outliers and can find non-linear clusters. It is a very powerful algorithm for outliers detection and clustering.

Attributes¶

After the object creation, all the parameters become attributes. The model will also create extra attributes when fitting the model:

Name Type Description
n_cluster
int
 Number of clusters created during the process.
n_noise
int
 Number of points with no clusters.

str
 Train relation.
X
list
 List of the predictors.
key_columns
list
 Columns not used during the algorithm computation but which will be used to create the final relation.
