Pokemon¶

This notebook is an example on how to use the VerticaPy Library. We will use the Pokemon dataset to predict which Pokemon will win on a 1v1 fight. You can download the Jupyter Notebook of the study by clicking here. To achieve our goals, we have access to the following data.

A table Pokemon with the information about the different Pokemons (Click here to download the dataset):

  • Legendary: True if it is a Legendary Pokemon
  • Sp_Atk: Special Attack
  • Generation: Pokemon Generation
  • Speed
  • Sp_Def: Special Defense
  • Name
  • Defense
  • Type_1: Pokemon first Type
  • Type_2: Pokemon second Type
  • HP: Number of life points
  • Attack

A table combats with information about the different fights (Click here to download the dataset):

  • First_pokemon: Pokemon of trainer 1
  • Second_pokemon: Pokemon of trainer 2
  • Winner: Winner of the fight

We will follow the entire Data Science cycle (Data Exploration - Data Preparation - Data Modeling - Model Evaluation - Model Deployment) and solve this easy use-case. The purpose is to show you some of the library interesting functionalities.

Features Engineering is the key and by computing the difference between pokemon 1 and 2 qualities, we can create amazing features. We can also change Winner to a binary signal (1 if the first pokemon won, 0 otherwise).

Machine Learning¶

Some really important features are categorical. Random Forest can handle them. Besides, we need trees which are deep enough to compare all the different types.

In [12]:
from verticapy.learn.ensemble import RandomForestClassifier
from verticapy.learn.model_selection import cross_validate

predictors = fights.get_columns(exclude_columns = ['Winner'])
response = "Winner"
relation = "pokemons_fights"
model = RandomForestClassifier("rf_pokemon", 
                               n_estimators = 50, 
                               max_depth = 100, 
                               max_leaf_nodes = 400, 
                               nbins = 100)
model.drop()
cross_validate(model, relation, predictors, response)
